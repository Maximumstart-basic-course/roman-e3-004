1.Что такое EcmaScript?

EcmaScrip это стандарт языка програмирования JS.

2.Для чего используются и чем отличаются атрибуты async и defer??

Даные атрибуты позволяют менять способ загрузки JS скрипта на странице. Аsync скрипт выполняется асинхронно, то есть, браузер продолжает обработку страницы а когда скрипт будет загружен – он выполнится. Defer очень схож, но с его помощью указываеться порядок подключения загруженых скриптов браузером, а в async первыми подключаються скрипты которые быстрее загрузились.

3.Почему предпочтительнее подключать скрипты перед </body>, а не в <head>...</head>???

Страница грузиться построчно, поетому тяжелый скрипт замедлит загрузку страницы и посетитель скорее уйдет....перед закрывающим тегом body рационально подключать скрипты которые не нужны для визуального восприятия контента а могут быть подгружены в конце.

4.Какие основные отличия интерпретируемого языка программирования от компилируемого?

В компилируемом языке код компилируеться с помощью ресурсов техники в машинный код. Данная стадия очень замедляет процес подгонки кода. В интерпретируемом языке код более прост для понимания и интерпритируеться в программе написания кода и выполняеться сразу.

5.Что такое языки высокого и низкого уровня? Какие ты знаешь??

Языки высокого уровня разработаны для прощего использования программистами (JavaScript, С++, PHP.). Языки низкого уровня имеют синтаксис очень близкий к машинному коду, они сложнее для изучение (Ассемблер).

6.Для чего используются переменные? Какие ключевые слова используются для создания переменных? Какие есть отличия в переопределении значений при задании переменной через let, const и var?

Переменная это способ обозначения значения для подальшей удобной работы. Var - способ обозначение перемен не принятый в ES6. Let - обозначение переменной которое будет меняться, к примеру в цыклах. Const обозначение для постоянной переменной.

7.Может ли имя переменной начинаться с цифры, символов _ или $??

Переменная не может начинаться с цифры, но с символов _ или $ может. Цифры в названии переменной могут использоваться.

8.Какие слова запрещено использовать в названии переменной?

Данные символы зарезервированы языком програмирования (var, class, return, export) и их нельзя использовать в качествете переменных.

9.Что такое camelCase и snake_case

camelCase это стиль написания слов где следующее слово пишеться слитно и с большой буквы, соответсвенно snake_case написание слов через нижнее подчеркивание.

10.Что такое "типы данных"? Какой оператор позволяет определить тип данных?

Существует 6 типов примитивных данных: number, string, boolean, null, undefined, object (ES6). Оператор который позволяет проверить тип даных typeof.

11.Что такое "динамическое приведение типов данных"? Можно ли писать JavaScript со статическим приведением типов?

Динамическое приведение типов данных в JS позволяет в одну и ту же переменную записывать значения различных типов, при этом тип самой переменной будет меняться. Можно.

12.Как привести строчное значение "10" к числу? Назовите два способа.

Первый вариант поставить унарный +, второй использовать команду Number(10).

13.Чем отличаются операторы "==" и "==="?

Ето операторы означают сравнение. "==" не строгое сравнени и "===" строгое сравнение.

14.Какие преимущества дают обратные кавычки (`) при указании строки.?

В отличии от обычных и двойных в них разрешён перевод строки и можно вставлять выражения при помощи ${…}.

1. Что такое "алгоритм"? Какие ты знаешь типы алгоритмов??

Алгоритм это четко определенная последовательность действий прописанная в коде. Линенйный, разветвленный, циклический.

2. Зачем нужны циклы? Какие виды циклов ты знаешь? В чем их основные отличия?

Циклы предусмотрены для многократного повторения одного участка кода. While, do...while, for, infinite loop. Главное отличие в пути выполнения цикла, и пути как он закончиться.

3. Что делают операторы ++i и i++ и в чем их отличие?

++i возвращает значение i после того, как оно было увеличено. i++ возвращает значение i до приращения.

4. Что такое "итерация"?

Итерация - повторение цикла.

5. Как называются и чем отличаются операторы || и &&?

|| оператор (ИЛИ), && оператор (И)

6. Для чего используется break? Почему в конструкции switch нужно использовать break для каждого case??

Оператор break нужен для прекращения case, в противном случаи цикл будет выполняться до конца.

7. Почему при выполнении указанного кода возникают ошибки?

(1) const i = 0; while (i <= 10) { console.log(i); }
(2) Переменная указана как const не подходит для цикла while (нужно let) и не задано условие выполнения(вечный цикл).

8. Почему при выполнении указанного кода возникают ошибки?

(1) for (j = 0; j <= 10; j++); { console.log(j); }
(2) Должно быть так: for (let j = 0; j <= 10; j++) { console.log(j); }

1. Зачем нужны функции??

Они позволяют повторять один и тот же код в разных частях программы.

2. Сколько параметров можно передать функции???

Параметров можно передать сколько необходимо.

3. Что случится, если передать в функцию меньше параметров, чем нужно? А если больше? Объясни на примере функции add(a, b).???

Если меньше то на месте параметра будет undefined и функция сработает. Если больше, то фунция тоже сработает на данные по неуказаным параметрам не отабражаться.

4. Возможно ли для параметра указать значение по умолчанию? Назови два способа?

1. Во внутренней переменной функции ставим = и пишем значение которое будет по умолчанию. 2. Использовать оператор ||.

5. Что такое область видимости??

В JS две области действия: глобальная и локальная. Переменная, объявляемая вне определения функции - глобальная. Ее значение доступно и может быть изменено в любом месте программы, а переменная которая объявлена внутри определения функции, является локальной.

6. Чем отличается Function Declaration от Function Expression??

Function Declaration – функция, объявленная в основном потоке кода. Function Expression – объявление функции в контексте какого-либо выражения, например присваивания.

7. Что такое замыкание??

Замыкание — это комбинация функции и лексического окружения, в котором эта функция была определена.
